## 📂 Active Laboratory Logs

### 🚀 [Phase 1: Bare-Metal Core Bootstrap & Bootloader Resolution](https://www.google.com/search?q=./arch-installation-logs/Part1-Core-Base.md)

* **Focus Area:** POSIX-compliant system bootstrapping, standard partition tables, and target file system topologies.
* **Core Tech Stack:** `x86_64` Architecture, GPT (`gdisk`), XFS Filesystem, `pacstrap` core utilities, and UEFI GRUB configuration.
* **Key Achievement:** Resolved standard boot configuration omissions (`grub rescue` state reconstruction via Live ISO recovery pipelines).

### 🖥️ [Phase 2: Display Server, Desktop Environment, & NVIDIA Graphics Integration](https://www.google.com/search?q=./arch-installation-logs/Part2-Nvidia-Plasma.md)

* **Focus Area:** High-performance desktop environment layering and proprietary monolithic hardware driver configuration.
* **Core Tech Stack:** KDE Plasma, Wayland/X11, NVIDIA Proprietary Driver Stack (`RTX 2070`), Kernel Mode Setting (KMS), and Systemd-SDDM.
* **Key Achievement:** Successfully troubleshot and eliminated software-fallback loop rendering (LLVMpipe) causing **95% CPU bottlenecking**, shifting compositor pipelines entirely onto GPU hardware structures (<2% CPU overhead).

### 🛠️ [Phase 3: Shell Script Automation (`setup.sh`)](https://www.google.com/search?q=./arch-installation-logs/setup.sh)

* **Focus Area:** Automated, rapid environment duplication and baseline package orchestration.
* **Core Tech Stack:** Bash POSIX Shell, Idempotent configuration logic, Linux Environment Variables, Security Profiles (`UFW`).
* **Key Achievement:** Programmed a secure, single-command system initializer that handles repository routing adjustments, Early KMS module insertion, dynamic environment masking, and automated local firewall policies without execution state drift.
